Event Bubbling – Summary

1. Definition
- Event bubbling is a way of propagating events in the DOM.
- When an event occurs on a child element, it first runs its own event listener, then propagates ("bubbles") upwards through its ancestors (parent → grandparent → document → window).

2. Flow
- Starts from: The deepest (target) element where the event happened.
- Then goes to: Parent → Parent’s parent → ... → document → window.
Example flow for a click inside a <button>:
button → div → body → html → document → window

3. Default Behavior
- By default, most DOM events (like click, keydown, submit) bubble unless specified otherwise.
Example:
document.querySelector("button").addEventListener("click", () => {
  console.log("Button clicked");
});
document.body.addEventListener("click", () => {
  console.log("Body clicked");
});
Clicking the button logs:
Button clicked
Body clicked

4. Event Object
- When the event bubbles, the same event object is passed to each ancestor listener.
- The object contains useful properties:
  target → where the event originally happened.
  currentTarget → the element whose listener is currently running.

5. Why Event Bubbling is Useful
- Event Delegation:
Instead of adding listeners to multiple child elements, add one listener to the parent and handle events based on event.target.
Example:
document.querySelector(".list").addEventListener("click", (e) => {
  if (e.target.tagName === "LI") {
    console.log("Clicked on:", e.target.textContent);
  }
});

- Centralized Event Handling:
You can handle events from multiple nested elements at one place.

6. Controlling Event Bubbling
- Stop Bubbling:
Use event.stopPropagation() to prevent the event from going further up.
Example:
child.addEventListener("click", (e) => {
  e.stopPropagation(); // event will not go to parent
});

- Prevent Default + Stop:
event.preventDefault() stops default browser action (e.g., link navigation), but does not stop bubbling.

7. Capturing vs Bubbling
- Bubbling is bottom → up.
- Capturing (or trickling) is top → down.
You can enable capturing with { capture: true }:
document.body.addEventListener("click", handler, { capture: true });

8. Real-World Examples
- Clicking inside a modal → event bubbles to document → can be used to detect outside clicks and close modal.
- Click inside a table row → handled by parent table to highlight row.

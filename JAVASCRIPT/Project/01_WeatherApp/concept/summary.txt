🟢 1️⃣ try...catch → Handles Network Errors

These happen when the request cannot reach the server.

Examples:

-No internet
-Wrong domain
-DNS failure
-CORS blocked
-Server completely unreachable

Example:

try {
  const response = await fetch(url);
} catch (error) {
  console.log("Network error:", error.message);
}


If network fails → Promise is rejected → goes to catch.

🔵 2️⃣ response.ok → Handles HTTP Errors

These happen when:

✔ Server is reached
✔ Server responds
❌ But response status is error (404, 500, 401 etc.)

Example:

const response = await fetch(url);

if (!response.ok) {
  throw new Error(`HTTP error: ${response.status}`);
}


Fetch does NOT reject for 404 or 500.

So you must manually check response.ok.

✅ Final Correct Version

There are three types of errors when using fetch():

1️⃣ JavaScript Runtime Error

Occurs before or outside the network request.

Example: variable not defined (ReferenceError)

Happens while building the URL or executing JS

Not related to HTTP or network

Thrown by the JavaScript engine

2️⃣ Network Error (Promise Rejected)

Occurs when the request cannot be completed.

Example: no internet, DNS failure, wrong domain, CORS blocked

fetch() promise is rejected

No response object is created

Error usually: TypeError: Failed to fetch

3️⃣ HTTP Error (Promise Fulfilled but Not OK)

Occurs when the server responds with an error status.

Example: 404, 401, 500

fetch() promise is fulfilled

response.ok === false

You must manually check and throw error

Example:

if (!response.ok) {
  throw new Error(data.message);
}

Mistake 1 :

Mistake:
Only checked clicked button.

Why:
Did not check other answer buttons.

Fix:
Loop through all buttons and find the correct one.

Rule:
Sometimes you must check all related elements, not just clicked one.

Mistake 2 :

Mistake:
Expected one button variable to represent all buttons.

Why:
answerBtn refers to only one button from forEach.

Fix:
When wrong answer happens, loop through answersBtn and compare each button text with correct answer.

Rule:
A variable from forEach represents one item, not the whole collection.

Why:

Fix:
Disable every answer button after selection.

Rule:

Mistake 6 :

Mistake:
Checked question limit outside click event.

Why:
Condition runs once at startup, not after index changes.

Fix:
Check limit after increasing index inside Next button logic.

Rule:
Conditions depending on changing values should run where value changes.


mistake : 

if (currentQuestionIndex < quizData.length) {
  nextBtn.addEventListener("click", () => {
    currentQuestionIndex++;
    showQuestion();
    console.log(currentQuestionIndex);
  });
} else {
  nextBtn.disabled = true;
}



nextBtn.addEventListener("click", () => {
  currentQuestionIndex++;
  if (currentQuestionIndex <= quizData.length - 1) {
    showQuestion();
  } else {
    nextBtn.disabled = true;
  }
});

Rule:
Conditions depending on changing values should run where value changes.

score = 4

Mistake 11:

Mistake:
Removed DOM using:

containerEl.innerHTML = ""

Why:
Question elements and buttons got deleted.

Flow:

Quiz End
↓
DOM nodes removed
↓
Restart
↓
showQuestion()
↓
nothing exists to update

Fix:

Use:

containerEl.style.display = "none";

Restart:

containerEl.style.display = "block";

Rule:
Hide UI when needed.
Do not destroy elements if you need them later.

-If elements are dynamic and created again later → innerHTML = "" can be okay.
-If elements are needed later for updates/reuse → hide/show is better.
---------------------------------------------------

Mistake 13 :

Mistake:
Used button text (nextBtn.textContent) to control app logic.

Why:
UI text can change anytime. Logic should not depend on displayed text because changing "Next" to "Continue" can break functionality.

Fix:
Use state variables or conditions based on data/state instead.

if (currentQuestionIndex < quizData.length)

or 

check using Boolean state variable 

Rule:
Never use UI text to control application logic. Use variables/state instead.


closure analogy(works)

BankAccount function scope (temporary)
  ├─ _balance = 20000
  └─ getBalance() { return _balance; }  <-- closure

After BankAccount finishes:
  └─ _balance "lives on" inside getBalance() and other methods

💡 In short:
A closure lets a function “remember” and access its parent function’s variables, even after the parent function has finished executing.

--------------
Closures only work inside functions because a closure is defined as:

A function that remembers the variables in the scope where it was created, even after that outer function has finished executing.

----------------
Conclusion:

this.getBalance = function() {…} creates a public method on the object instance, accessible outside.

A normal function getBalance() {…} inside the constructor is private and cannot be called on the instance.

Using this allows the instance to expose methods while still keeping variables like _balance private via closures.

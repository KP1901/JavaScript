BOM InterView Questions

1. What is the Browser Object Model (BOM) and why is it important?
Answer: BOM allows JS to interact with the browser environment (not just the page). It is important because it lets developers manipulate windows, URLs, history, screen size, and other browser features programmatically.
2. How is BOM different from DOM?
Answer: DOM manipulates the HTML content (document). BOM manipulates the browser (window, history, location). Example: changing URL without reload uses BOM; updating text on the page uses DOM.
3. What is the window object in BOM?
Answer: The global object representing the browser. All global JS variables and functions are attached to window. It includes dialogs, timers, and browser properties.

Answer: Using screen.width, screen.height, screen.availWidth.
Practical: Adjust layout for responsive design or full-screen apps.
7. How do you use the location object?

9. Explain alert, confirm, and prompt.
Answer: Dialog methods that block code execution.
alert() → message
confirm() → returns true/false
prompt() → returns user input

11. Explain setInterval and its use.
Answer: Executes a function repeatedly at fixed intervals. Must be cleared using clearInterval(id) to avoid infinite loops.
12. Difference between setTimeout and setInterval.
Answer: setTimeout → once; setInterval → repeated. Recursive setTimeout is often better to prevent overlaps in long-running functions.
13. How do you stop timers in BOM?

clearInterval(intervalId);

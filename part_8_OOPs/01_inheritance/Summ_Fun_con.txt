"Yes, that's correct. When you create an instance like let kiran = new Employee(), JavaScript sets up the prototype chain as follows:

kiran.__proto__ === Employee.prototype;
Employee.prototype.__proto__ === Person.prototype;


This means kiran can access properties and methods defined on both Employee.prototype and Person.prototype. To establish this chain, we use:

Employee.prototype = Object.create(Person.prototype);


This line creates a new object whose prototype is Person.prototype and assigns it to Employee.prototype. Now, instances of Employee (like kiran) can access methods defined on Person.prototype through the prototype chain."

Key Points:

Prototype Chain: When kiran is created, its internal [[Prototype]] points to Employee.prototype. Through Employee.prototype, it can access Person.prototype due to the chain established by Object.create(Person.prototype).

Object.create(): This method creates a new object with the specified prototype object and properties. In this case, it creates an object with Person.prototype as its prototype, which is then assigned to Employee.prototype.

Constructor Functions: Every function in JavaScript has a prototype property. When you define a constructor function like Employee, it automatically gets a prototype property, which is used to set up the prototype chain for instances created with that constructor.

🧠 How this.salary Works

Constructor Function: When you create an instance using new Employee('Kiran', 50000), the Employee function is invoked with this referring to the new object being created. This sets kiran.name to 'Kiran' and kiran.salary to 50000.

Prototype Method: The getSalary method is added to Employee.prototype. This means it's available to all instances of Employee, including kiran.

Method Invocation: When you call kiran.getSalary(), JavaScript looks for the getSalary method on kiran. Since it's not found directly on kiran, JavaScript checks kiran.__proto__, which points to Employee.prototype, and finds the method there.

Accessing this.salary: Inside getSalary, this refers to kiran because that's the object on which the method was called. Therefore, this.salary accesses kiran.salary, which is 50000.
